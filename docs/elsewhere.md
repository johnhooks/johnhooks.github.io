# Elsewhere

Elsewhere is the site's optional doorway into the strange. The main site remains a clean,
text-strong place to publish writing. Elsewhere is where the site can become exploratory,
colorful, procedural, and deliberately odd.

This is planned design direction, not current behavior.

## Purpose

Elsewhere should create a second mode for the site without compromising the first one.

The default site says: here is the writing.

Elsewhere says: here is the website thinking, mutating, and wandering through itself.

It should invite discovery rather than replace normal navigation. A reader should be able
to ignore Elsewhere entirely and still have a complete experience of the blog.

## Design direction

Elsewhere is inspired by early 2000s personal web exploration, including bold styling,
maze-like navigation, strange layouts, and pages that felt authored by one person rather
than generated by a product template.

The goal is not nostalgia cosplay. Elsewhere should use modern CSS and web platform
features to make something current, fast, and maintainable.

The visual language can include:

- bold, saturated colors
- stark contrast
- full-page compositions
- odd type scale shifts
- overlapping or rotated elements
- intrinsic layouts that morph across viewport sizes
- quotes, images, excerpts, and fragments
- procedural placement, color, and motion
- page transitions that make movement feel like a journey

## Relationship to the main site

The main site should keep its existing design principles: narrow reading measure,
restrained motion, authored CSS, semantic HTML, and text-first presentation.

Elsewhere is allowed to break those rules because it is explicitly entered as a different
mode. That boundary matters. It protects the blog while giving the site room to
experiment.

Shared content can flow into Elsewhere, but the layout and browsing behavior should be
separate from the default post and project templates.

## CSS principles

Elsewhere should push CSS custom property inheritance as a central design tool. Pages and
components can define local variables for color, scale, spacing, animation timing,
rotation, density, and contrast. Child elements should inherit and reinterpret those
variables instead of relying on large one-off style blocks.

This keeps the weirdness systemized without making it feel generic.

Possible variables include:

```css
--elsewhere-bg
--elsewhere-fg
--elsewhere-accent
--elsewhere-secondary
--elsewhere-scale
--elsewhere-density
--elsewhere-angle
--elsewhere-motion-duration
--elsewhere-motion-ease
```

Layouts should favor intrinsic behavior over fixed breakpoints where practical. The
composition should change meaningfully as available space changes, not simply stack from
desktop to mobile.

## Motion and transitions

Motion can be more expressive in Elsewhere than on the main site. It can use animation,
view transitions, hover states, and procedural movement to create a sense of travel.

Motion must still respect `prefers-reduced-motion`. Reduced motion should preserve the
visual identity while removing unnecessary movement.

## Content model

Elsewhere should reuse existing site content where possible:

- post titles
- excerpts
- quotes
- tags
- dates
- images
- project names
- code fragments
- selected manually curated material

The first versions should avoid creating a heavy second publishing workflow. Optional
metadata can be added later if it proves useful.

## First project

The first Elsewhere project should be a procedural journey through programming quotes.

It should be small enough to build and iterate on, but expressive enough to establish the
Elsewhere design language: bold color, full-page layouts, procedural variation, strange
navigation, and modern CSS-driven motion.
